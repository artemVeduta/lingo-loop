## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

Document PASS/FAIL for each gate. Any FAIL requires an entry in Complexity
Tracking with the simpler alternative rejected.

- **Layered boundaries**: Plan identifies affected layer(s): host adapter, core,
  DAL, renderer, skills, hooks, or packaging. Cross-layer calls are immediate
  neighbor calls only and are justified.
- **Contracts and abstractions**: Changed data crosses boundaries through
  Pydantic models, JSON schemas, narrow Protocols, SQL migrations, or documented
  CLI JSON. No catch-all dictionaries or concrete DAL/adapter dependencies leak
  across layers.
- **Deterministic tests**: Plan lists required unit, golden, contract,
  integration, migration, and semantic-eval coverage for changed behavior.
- **Local-first data ownership**: YAML remains human-editable config only;
  SQLite remains transactional/derived state only. Paths are platform/XDG based.
- **Scope discipline**: Work is limited to current v1 requirements unless this
  feature explicitly amends scope. No speculative hosts, modalities,
  dependencies, or methods without current callers.
- **DRY and composition**: Shared schemas, tag vocabularies, severity maps, path
  rules, and prompt rubrics stay single-source. Behavior is composed through
  explicit collaborators, not inheritance or global mutable state.
